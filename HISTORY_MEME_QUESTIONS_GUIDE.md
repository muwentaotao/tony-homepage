# 历史梗挑战题目维护说明

题目统一维护在 `src/data/historyMemeQuestions.js`。添加、删除或修改题目时，只需要编辑这个文件，不需要修改游戏页面。

## 添加新题时需要提供的内容

添加新题时，只需要提供以下三项内容：

- `question`：玩家看到的题目。
- `correctAnswer`：唯一正确答案。
- `explanation`：答对后显示的简短说明，建议一到两句话。

Codex 会按现有顺序补充唯一 `id`，并为每道题补充正好 5 个 `wrongAnswers`。

`tag` 已不再需要，页面不会显示或依赖该字段。旧题目中已有的 `tag` 可以暂时保留，新题目无需填写。

## 示例题格式

```js
{
  id: 'hmq-006',
  question: '“烽火戏诸侯”这个大型失信现场，传统故事里是哪位君王的操作？',
  correctAnswer: '周幽王',
  wrongAnswers: ['秦穆公', '齐桓公', '楚庄王', '晋文公', '周厉王'],
  explanation: '传统叙事中，周幽王为博褒姒一笑点燃烽火，后来真有外敌时诸侯不再相信。',
},
```

新增题目时，把完整的题目对象放进 `historyMemeQuestions` 数组中，并注意前后题目之间需要用英文逗号分隔。

## 错误选项生成原则

- 必须正好有 5 个错误选项。
- 错误选项不能包含正确答案。
- 每个错误选项之间不能重复。
- 尽量选择同一时代、同一身份或容易混淆的人物与事件，让题目有辨识度但不故意误导。
- 避免有争议、存在多个正确答案或依赖冷僻说法的选项。

游戏会在每轮开始时自动把 `correctAnswer` 和 `wrongAnswers` 合并并随机打乱，不需要手动调整答案位置。

## 题目数量

题目数量不固定。当前题目文件里有几道题，一轮挑战就会使用几道题；不需要在其他文件中填写题目总数。

## 修改后如何本地测试

1. 在项目目录运行 `npm run dev`。
2. 打开终端中显示的本地地址，进入“历史梗挑战”。
3. 确认题目、6 个选项和答对后的说明正常显示。
4. 多开始几轮，确认题目顺序和选项顺序会随机变化。
5. 查看浏览器开发者工具的控制台。如果出现 `[历史梗题库]` 警告，请按提示修正重复 `id` 或错误选项数量。
6. 最后运行 `npm run build`，确认项目可以正常构建。
